# 카피 — 대시보드 (Figma 개편 기준 · 최종 확정)

> 톤: 숫자가 주인공, 카피는 받쳐주는 역할. 짧게. 첫 등장 용어만 풀이.
> 보이스 우선순위: **신뢰 > 친근 > 동기부여**. 금지 표현은 `docs/brand/branding-guide.md` 8항 준수.
> **변경(2026-06-28)**: SLA 점수 제거(완료율 프록시였음) → **수락률 게이지가 메인 지표**. Figma 개편 반영.
> 서비스명 placeholder: `배달장부(테스트)` — 확정 시 일괄 치환(별도 메모 참조).

## 1. 상단 헤더

- **인사말**: `{이름}님, 오늘도 안전 운행해요!`
  - `{이름}`만 굵게(bold), 나머지는 일반. 느낌표 1개까지만(과한 텐션 금지).
- **실시간 상태 인디케이터**: `● 실시간 업데이트 중`
  - 점(`●`)은 라이브 상태색(uxui 토큰). 갱신 끊김 시 문구 분기 → §6 참조.
- **날짜·시각 표기**: `26년 6월 28일 (일) 오후 2:08`
  - 포맷: `YY년 M월 D일 (요일) 오전/오후 h:mm`. 요일은 한 글자(일·월·화…). 분은 2자리.
  - 보조 안내(작게, 선택): `기록은 1분마다 자동으로 새로워져요`

## 2. 기간 탭

| 탭 라벨 | 짧은 라벨(공간 부족 시) | 기준 |
|---------|------------------------|------|
| `오늘` | `오늘` | 오늘 0시부터 지금까지 |
| `이번 주` | `주` | 이번 주 월요일부터 |
| `이번 달` | `월` | 이번 달 1일부터 |

- 선택된 탭에 따라 §3 메인 라벨이 바뀐다(오늘→이번 주→이번 달).

## 3. 메인 지표 — 수락률 게이지

> 화면 중앙 주인공. 게이지 안/옆에 큰 퍼센트 숫자.

- **라벨(기간 연동)**:
  - 오늘 탭: `오늘 수락률`
  - 이번 주 탭: `이번 주 수락률`
  - 이번 달 탭: `이번 달 수락률`
- **값 표기**: `{n}%` (게이지 중앙, 숫자가 가장 큼)
- **한 줄 설명(i 아이콘 탭 시)**: `들어온 콜 중 내가 받은 비율이에요. (받은 콜 ÷ 전체 콜)`
- **값에 따른 보조 문구**(게이지 하단, 1줄·조건부):
  - 높음(≥90%): `콜을 잘 잡고 있어요`
  - 보통(70~89%): *(보조 문구 생략 — 숫자만)*
  - 낮음(<70%): `이번 {기간}엔 조금 낮아요` *(질책 금지 — 사실+여지만. "{기간}"=오늘/이번 주/이번 달)*

## 4. 통계 — 완료 / 거절 / 취소

> 게이지 아래 3개 카드(또는 행). 라벨 + 큰 숫자 + 단위.

| 지표 | 라벨 | 값 표기 | 한 줄 설명(i 탭 시) |
|------|------|---------|---------------------|
| 완료 | `완료` | `{n}건` | `정상적으로 끝낸 배달 건수예요.` |
| 거절 | `거절` | `{n}건` | `들어온 콜을 받지 않은 건수예요.` |
| 취소 | `취소` | `{n}건` | `배차 후 취소된 건과 픽업 후 취소된 건을 합한 수예요.` |

- **0건 처리**: 값이 0이어도 `0건`을 그대로 노출(숨기지 않음). 오류가 아니라 정상.
  - 필요 시 보조: `이 기간엔 {지표} 기록이 없어요` *(0건 안심)*
- **취소 발생 시 보조(선택)**: `취소 {n}건이 잡혔어요` *(원인 단정·책임 추궁 금지)*
- ⚠️ **backend/frontend 확인 필요**: 기존 `배차취소`+`배달취소`가 단일 `취소`로 합산됨. 합산 로직 일치 여부 확인.

## 5. 피크타임 완료 현황

- **섹션 제목**: `피크타임 완료 현황`
- **항목 라벨**(Figma 원안 → 다듬은 표기 제안):

| Figma 원안 | 확정 라벨(권장) | 값 표기 |
|------------|------------------|---------|
| 아침점심피크 | `아침·점심 피크` | `{n}건` |
| 오후논피크 | `오후 비피크` | `{n}건` |
| 저녁피크 | `저녁 피크` | `{n}건` |
| 심야논피크 | `심야 비피크` | `{n}건` |

- 다듬은 이유: `논피크`(non-peak)는 외래어 혼용 → 한글 `비피크`로 통일, 가독성↑. **만약 라이더가 현장에서 "논피크"를 일상어로 쓰면 원안 유지 가능**(현장 언어 존중 원칙). frontend·uxui와 최종 합의.
- **시간대 툴팁(선택, 데이터 구간 정의 확정 시)**: `{시작}~{끝}시에 완료한 배달이에요` (24시간제 표기)
- **빈 구간 안내**: `이 시간대는 아직 기록이 없어요`
- **하이라이트(선택)**: `{구간}에 가장 많이 달렸어요`

## 6. 실시간 상태 분기 (헤더 인디케이터)

| 상태 | 인디케이터 문구 |
|------|------------------|
| 정상 | `● 실시간 업데이트 중` |
| 갱신 지연(스크래퍼 지연) | `● 갱신이 조금 늦어지고 있어요` |
| 연결 끊김 | `● 잠시 연결이 끊겼어요` |

## 7. 새로고침

- **버튼**: `새로고침`
- **갱신 완료 토스트**: `최신 기록으로 새로워졌어요`
- **갱신 중 표시(선택)**: `불러오는 중…`

## 8. 드로어 메뉴

| 항목 | 라벨 | 상태 | 보조 라벨(권장) |
|------|------|------|------------------|
| 내 정보 | `내 정보` | 비활성 | 배지 `준비 중` |
| 랭킹 | `랭킹` | 비활성 | 배지 `준비 중` |
| 로그아웃 | `로그아웃` | 활성 | — |

- **비활성 항목 탭 시 토스트**: `아직 준비 중이에요. 곧 열릴게요.`
- **로그아웃 확인**: `로그아웃할까요? 다음에 다시 카카오로 로그인하면 돼요.` / CTA `로그아웃` · `취소`
- ⚠️ **`랭킹` 보이스 충돌 경고(lead 보고)**: 브랜딩 가이드 금지표현 #2(순위·경쟁 압박) 및 "타 라이더 비교 금지" 원칙과 정면 충돌. 현재 비활성이라 당장 문제는 없으나, **실제 오픈 시점에 반드시 재검토** 필요. 대안: (a)타인 서열이 아닌 "과거의 나 vs 지금의 나" 자기기록 갱신(예: `내 기록`/`내 베스트`), (b)명칭/구성 변경. 경쟁·서열 노출형 랭킹은 보이스상 비권장.

## 9. 로딩 / 에러 / 빈 상태

### 로딩
- 최초 로딩: `내 기록을 불러오는 중…`
- 갱신 로딩: `불러오는 중…`

### 빈 상태(데이터 없음)
- **오늘 기록 없음**
  - 제목: `아직 오늘 기록이 없어요`
  - 본문: `첫 콜을 받으면 여기에 실적이 채워져요. 기록은 1분마다 자동으로 새로워져요.`
  - CTA: `새로고침`
- **이번 주/이번 달 기록 없음**: `이번 {기간}엔 아직 기록이 없어요.`
- **계정 연결됐으나 동기화 전**: `기록을 불러오는 중이에요. 잠깐만 기다려 주세요.`

### 에러
- **불러오기 실패(서버)**
  - 제목: `잠깐 문제가 생겼어요`
  - 본문: `기록을 불러오지 못했어요. 잠시 후 다시 시도해 주세요.`
  - CTA: `다시 시도`
- **네트워크 오류**
  - 제목: `연결이 불안정해요`
  - 본문: `인터넷 연결을 확인하고 다시 시도해 주세요.`
  - CTA: `다시 시도`
- **세션 만료**
  - 제목: `다시 로그인이 필요해요`
  - 본문: `안전을 위해 로그인이 만료됐어요. 카카오로 다시 로그인하면 이어서 볼 수 있어요.`
  - CTA: `다시 로그인하기`

> 상세 에러 전체 목록은 `docs/copy/errors.md` 참조. 위는 대시보드에서 자주 만나는 케이스만.
