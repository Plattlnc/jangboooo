# 카피 — 인증 (카카오 로그인 · 본인인증 · 계정 연결)

> 톤: 신뢰 최우선. "왜 이 정보를 받는지"를 먼저 안심시킨다.

## 1. 카카오 로그인 화면 (첫 진입)

- **서비스 한 줄 소개(히어로)**: `내 배달 성적표, 내 손안에.`
- **보조 설명**: `완료·수락률·취소까지, 내 실적을 오늘·주·월로 한눈에 봐요.`
- **로그인 버튼(CTA)**: `카카오로 시작하기`
- **버튼 하단 안심 문구**: `카카오 계정으로 안전하게 로그인해요. 친구 목록이나 메시지는 가져오지 않아요.`
- **약관/개인정보 동의 라인**: `시작하면 [이용약관]과 [개인정보 처리방침]에 동의하게 돼요.`

## 2. 본인인증(휴대폰) 안내

> 목적: 휴대폰 번호로 관리시스템 명단과 매칭해 내 데이터를 찾아 연결한다. 라이더에겐 "왜 번호가 필요한가"가 핵심 불안.

- **화면 제목**: `휴대폰으로 내 기록을 찾을게요`
- **설명 본문**:
  `등록된 휴대폰 번호로 내 배달 기록을 안전하게 연결해요.`
  `번호는 본인 확인과 기록 연결에만 쓰이고, 따로 저장해 마케팅에 쓰지 않아요.`
- **인증 시작 CTA**: `휴대폰 인증하기`
- **인증번호 입력 안내**: `문자로 받은 6자리 숫자를 입력해 주세요.`
- **인증번호 입력 placeholder**: `인증번호 6자리`
- **재전송 버튼**: `인증번호 다시 받기`
- **재전송 타이머**: `{초}초 후에 다시 받을 수 있어요`
- **확인 CTA**: `인증 완료하기`

### 안심 문구 (본인인증 화면 하단)
- `이 점수와 기록은 평가가 아니라 내 기록이에요. 정산이나 배차에 영향을 주지 않아요.`

## 3. 계정-데이터 연결 (매칭 결과)

### 매칭 성공
- **제목**: `내 기록을 찾았어요`
- **본문**: `{라이더명}님의 배달 기록이 연결됐어요. 지금 바로 확인해 봐요.`
- **CTA**: `내 대시보드 보기`

### 매칭 실패 (명단에 번호 없음)
- **제목**: `연결할 기록을 못 찾았어요`
- **본문**: `이 번호로 등록된 라이더 기록이 없어요. 관리 담당자에게 등록된 번호가 맞는지 확인해 주세요.`
- **보조**: `번호를 바꿔 다시 시도할 수도 있어요.`
- **CTA(주)**: `다른 번호로 인증하기`
- **CTA(보조)**: `문의하기`

## 4. 로그아웃 확인
- **문구**: `로그아웃할까요? 다음에 다시 카카오로 로그인하면 돼요.`
- **CTA**: `로그아웃` / `취소`
